## 🔍 search_knowledge 节点内部流程（并行检索）

```
┌─────────────────────────────────────────────────────────────────────────┐
│              search_knowledge Node - Parallel Hybrid Search             │
└─────────────────────────────────────────────────────────────────────────┘

query: "RAG 检索机制"
   ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  ToolRegistry.execute(QUERY_KNOWLEDGE_HUB)                               │
└──────────────────────────────────────────────────────────────────────────┘
   ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  HybridSearch.search()                                                    │
│                                                                          │
│  Step 1: Query Processing                                                │
│  ┌────────────────────────────────────────┐                              │
│  │ QueryProcessor                         │                              │
│  │ - 提取关键词：["RAG", "检索", "机制"]   │                              │
│  │ - 提取 filters                         │                              │
│  └────────────────────────────────────────┘                              │
│                                                                          │
│  Step 2: Parallel Retrieval (ThreadPoolExecutor)                        │
│  ┌───────────────────┐       ┌───────────────────┐                      │
│  │  DenseRetriever   │       │  SparseRetriever  │                      │
│  │                   │       │                   │                      │
│  │  - 语义相似度     │       │  - BM25 关键词     │                      │
│  │  - Embedding      │       │  - 词频统计        │                      │
│  │  - 向量检索       │       │  - 倒排索引        │                      │
│  │                   │       │                   │                      │
│  │  ThreadPoolExecutor (并行执行，互不阻塞)       │                      │
│  └─────────┬─────────┘       └─────────┬─────────┘                      │
│            │                           │                                 │
│            │ 返回 top_k=20             │ 返回 top_k=20                   │
│            ↓                           ↓                                 │
│  ┌─────────────────────────────────────────────────────────┐            │
│  │              RRF Fusion (Reciprocal Rank Fusion)        │            │
│  │                                                          │            │
│  │  score = 1 / (k + rank_dense) + 1 / (k + rank_sparse)   │            │
│  │                                                          │            │
│  │  - 合并两个结果集                                         │            │
│  │  - 计算 RRF 分数                                          │            │
│  │  - 重新排序                                              │            │
│  └─────────────────────────────────────────────────────────┘            │
│                     ↓                                                    │
│  ┌─────────────────────────────────────────────────────────┐            │
│  │           Post-Filtering (元数据过滤)                    │            │
│  │  - 应用 collection 过滤                                   │            │
│  │  - 应用其他 metadata 约束                                 │            │
│  └─────────────────────────────────────────────────────────┘            │
│                     ↓                                                    │
│  ┌─────────────────────────────────────────────────────────┐            │
│  │              Final Results (top_k=10)                   │            │
│  └─────────────────────────────────────────────────────────┘            │
│                                                                          │
│  返回详细统计：                                                           │
│  - dense_count: 20                                                       │
│  - sparse_count: 20                                                      │
│  - fusion_method: "RRF"                                                  │
│  - parallel_execution: true                                              │
```

## 📝 完整执行轨迹示例

### 查询："RAG 有什么优势？"

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Execution Trace (执行轨迹)                                              │
└─────────────────────────────────────────────────────────────────────────┘

Step 1: classify_intent
├─ thought: "分析用户意图：RAG 有什么优势？..."
├─ action: "IntentClassifier.classify()"
├─ observation: "Intent: query (confidence: 0.95)"
└─ metadata: {intent: "query", confidence: 0.95, parameters: {...}}

Step 2: route
├─ thought: "根据意图路由到对应处理节点"
├─ action: "route_by_intent(query)"
└─ observation: "路由到 search_knowledge"

Step 3: search_knowledge ⚡ (并行检索)
├─ thought: "并行检索：Dense(语义) + Sparse(BM25) → RRF 融合"
├─ action: "hybrid_search(query='RAG 有什么优势？', top_k=10)"
├─ observation: "并行检索完成 | Dense: 20 | Sparse: 20 | 融合后：10"
└─ metadata: {
       dense_count: 20,
       sparse_count: 20,
       fusion_method: "RRF",
       parallel_execution: true,
       result_count: 10
   }

Step 4: generate_response
├─ thought: "整理结果并生成最终回复"
├─ action: "format_response"
├─ observation: "生成查询回复 (10 条结果)"
└─ metadata: {formatted: true}

最终回复:
"为你找到 10 条与「RAG 有什么优势？」相关的内容..."
```

## 🎯 关键设计亮点

### 1. 并行检索（已有功能）
- ✅ `ThreadPoolExecutor` 同时执行 Dense + Sparse
- ✅ RRF Fusion 等待两个检索完成后融合
- ✅ 优雅降级：一个失败时使用另一个

### 2. 状态机可视化
- ✅ LangGraph 提供清晰的状态流转图
- ✅ 每个节点独立，易于调试
- ✅ 条件路由替代 if-else

### 3. 详细执行轨迹
- ✅ 每个节点记录 thought, action, observation
- ✅ metadata 包含并行检索统计
- ✅ 支持 observability 和 debugging

### 4. 上下文感知
- ✅ ConversationMemory 管理多轮对话
- ✅ 指代消解："它" → "RAG"
- ✅ 查询改写：隐含意图明确化

## 📈 性能优势

```
传统串行检索:
┌──────────┐    ┌──────────┐    总耗时
│  Dense   │ →  │  Sparse  │ = t1 + t2
└──────────┘    └──────────┘

LangGraph 并行检索:
┌──────────┐
│  Dense   │ ──┐
└──────────┘   │    总耗时
               ├→  RRF  → max(t1, t2) + t_fusion
┌──────────┐   │
│  Sparse  │ ──┘
└──────────┘

性能提升：~40-50% (当 t1 ≈ t2 时)
```
